Bolt vs Lovable vs Same.new

TLDR: Three working birding CRUD apps built and deployed in 10 minutes across these platforms

Prototyping delivers that immediate dopamine hit when showing colleagues or friends something tangible. There's genuine motivation in seeing results quickly, which makes rapid prototyping tools incredibly appealing for validating ideas over a weekend.

AI has obviously accelerated prototyping, but significant friction remains. Andrej Karpathy highlighted these exact frustrations in his recent Y Combinator talk "Software Is Changing (Again)" about Software 3.0, pointing out how even with advanced AI tools, developers still spend enormous time on deployment, database setup, and API integrations that require manual browser clicking rather than intelligent automation. His example showed building a working app demo in hours, then spending a week making it production-ready through tedious DevOps work.

That talk was a month ago, which when converted to the 2025 time-scale is more like 5 years ago, and there are already tools claiming to solve these problems. So I built my weekend project using three AI app builders: Lovable, Bolt, and Same.

Here's how that went. I had three working birding apps with a Supabase authentication and database in 10 minutes flat.

Getting Started with Integrations

All three platforms offer Supabase integration, which eliminates table setup and project configuration using the Supabase MCP.

Each tool created a Supabase project directly from the chat interface and configured tables automatically.

Here's how easy it was to integrate Supabase with each tool:

Lovable had great UX. The chat UI allowed me to select Supabase as a tool, letting the model know I wanted this to have a Supabase backend so the AI would not mistakenly setup a SQLite database or something like that, burning through tokens unnecessarily. The setup is a few button clicks to link to a Supabase account and then the AI handles the rest from project creation to table setup and migrations.

Bolt.new similarly has an easy Supabase integration through the setting. After prompting the AI intelligently suggests creating database with Supabase even if I have not explicitly asked for a Supabase integration. I prefer the Lovable UI that lets me explicitly suggest Supabase in the chat.

Same.new had some UX issues. I was not able to set up Supabase before starting a chat so the first time I prompted the app it went straight ahead and built the app using a SQLite database. To get around this I started over and prompted the AI to wait for me to integrate Supabase before continuing. From here the integration was straightforward but I burned through some unnecessary tokens unfortunately.

The One-Shot Test

I prompted each tool to make the same app:

Build a birding app where users can search for birds using the iNaturalist API, add birds to their personal sighting list with photos and notes, and authenticate to save their data. Users should be able to upload photos of their bird sightings to the database.

Unbelievably, each tool had finished the application, (which was fully functional! Strange birding API and all) in 10 minutes. I was stumped. Have a look at the results.

Lovable delivered a perfectly functional application from a single prompt. Authentication worked correctly, CRUD operations handled bird sightings smoothly, and the iNaturalist API integration functioned flawlessly. The design remained simple but everything worked across mobile and desktop browsers.

Bolt.new also produced a working prototype in one attempt. The green design with bird iconography felt cohesive, mobile responsiveness worked well, and core functionality operated reliably. The only noticeable issue being the tiny bird images in search results—a minor UX oversight.

Same.new created a fully functional app with a design that looked impressive with a beautiful landing page, working search functionality for non-authenticated users, and awesome hover animations. Lovable and Bolt both hid all style and functionality behind the login screen, where as Same seems to have thought a lot about selling the app as a product with a front page that had some features and information for user's to try out before signing up.

Vibe Check

These tools were incredible to use. The boundary for non-technical users to create something useful is disappearing at such a rate. I feel like I could suggest any of these to the next family member asking for help with their app idea.

I enjoyed both Bolt and Lovable's thoughtful UI design. The way they guide you through integrations and handles the development workflow feels polished and intentional.

The styling from Same.new felt premium rather than prototyped. It felt like prototyping is changing from functional demos to something someone would actually want to use.

Some quality of life improvements and adopting more extensive MCP tooling could make these really powerful, multi-purpose app building platforms.
